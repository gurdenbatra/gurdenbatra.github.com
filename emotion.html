Emotions in Speech

Ideation, Prototyping, Coding

September 2016

The goal of the project was to analyse emotions in human speech using Pure Data and machine. The inspiration comes from the fact that we depend on machines for various things and interactions. As machines get “smarter” by the day, it is important that they have some processing power to understand how the human users are feeling and respond appropriately.

The interesting part is that we can see the different emotions flashing and changing on the screen while the machine learning algorithm is trying to detect which emotion it is. As it is impossible to say we have only one objective emotion at a given time, it is mostly a mix of different emotions.






Sound is analyzed using four properties-

  • Bonk - Measures the attacks in the sound and the text speed.
  • Fiddle - Measures the pitch of the sound.
  • Energy - Mesures attack and decay of sound using FFT.
  • Centroid - Measure where the center of mass of the spectrum of sound, in a way the “brightness” of the sound.











Wekinator is used for machine learning analysis of sound and determining which emotion it has. It has been trained using an open source database of human sounds corresponding to different emotions.

Sound is taken as input from mic and the previously mentioned four properties are packed together and sent to Wekinator using OSC. Wekinator then returns an integer representing the emotion which is translated to text and printed on the GEM graphic object as the output. The output is one of the following emotions- anger, anxiety, boredom, disgust, happiness, neutral, and sadness.
