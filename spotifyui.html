Spotify Tangible Interface

Ideation, Prototyping, Coding

December 2016

The goal of the project was to create a tangible interface which physicalizes Spotify music listening experience. And to experiment and understand the MYO gesture band and Velostat(pressure sensitive material).

The result was the following prototype-






MYO Gesture Band

The MYO band has an app store like market already with connectors for a lot of apps including Spotify. But unfortunately, the default SDK for MYO doesn't work as advertised. There are lots of small issues like constantly getting locked, needing calibration and bad gesture recognition.

Hence I decided to implement by own gestures for the MYO band using a machine learning tool called Wekinator. Using this library, all my hand/arm movements from MYO were being sent to Wekinator using OSC port as a 10-dimensional vector composed of accelerometer, gyroscope, and orientation data. Using Wekinator's dynamic time wrapping gesture learning technique, I taught it to identify two different moving gestures and a resting state as well.

When Wekinator has learnt the gestures and is in running state it sends signals when it recognizes a gesture using OSC to a NodeJS web app which receives OSC and then controls Spotify using AppleScript. The simple code is here.








Effects Interface

To make the digital music physical and tangible, I wanted to give the user an external control to the music. To modify the music being played in Spotify, the audio is passed from Spotify to Pure Data using Soundflower(MacOS system extension to pass audio from one application to other) and then audio out through a USB audio interface.

The hardware interface has two velostat layers taped to each other, both with copper strips over them. One copper strip goes to the voltage out of an Arduino Uno and the other strip goes to ground and to an analog read slot of the Arduino through a resistor. This layered and taped velostat is sandwiched inside two wooden planks and this circuit measures how much pressure the user is applying on the wooden surface. On top the wooden surface, there is membrane potentiometer connected to the Arduino voltage out, ground and another analog read. It enables to read the user's exact position on the interface. The wood and the box for Arduino were made using a laser cutter.

In Pd, using the Firmata patch, position read from the membrane potentiometer has been mapped with increasing/decreasing the pitch of the audio signal and pressure read of the velostat has been mapped to increase/decrease reverb effect of the audio signal. This makes the Spotify music easily manipulatable and physical.
